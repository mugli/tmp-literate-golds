sortByCommitDateDesc returns the passed commits, sorted by `committer.When desc` Following this strategy, it is tried to reduce the time needed when walking the history from one commit to reach the others. It is assumed that ancestors use to be committed before its descendant; That way `Independents(A^, A)` will be processed as being `Independents(A, A^)`; so starting by `A` it will be reached `A^` way sooner than walking from `A^` to the initial commit, and then from `A` to `A^`.
