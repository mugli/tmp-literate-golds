In the future it's possible that compiler optimizations will make these unsafe operations unnecessary: https://golang.org/issue/2205. Both of these wrapper functions still incur function call overhead since they will not be inlined. We could write Go/asm copies of Sum64 and Digest.Write for strings to squeeze out a bit more speed. Mid-stack inlining should eventually fix this.
