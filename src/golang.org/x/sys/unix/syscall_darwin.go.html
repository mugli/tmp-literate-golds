The parameters for the OS X implementation vary slightly compared to the linux system call, specifically the position parameter: linux: int setxattr( const char *path, const char *name, const void *value, size_t size, int flags ); darwin: int setxattr( const char *path, const char *name, void *value, size_t size, u_int32_t position, int options ); position specifies the offset within the extended attribute. In the current implementation, only the resource fork extended attribute makes use of this argument. For all others, position is reserved. We simply default to setting it to zero.
