Error contains an error response from the server.
Code is the HTTP response status code and will always be populated.
Message is the server response message and is only populated when explicitly referenced by the JSON server response.
Details provide more context to an error.
Body is the raw response returned by the server. It is often but not always JSON, depending on how the request fails.
Header contains the response header fields from the server.
	Header http.Header

	Errors []ErrorItem
}
ErrorItem is a detailed error code & message from the Google API frontend.
Reason is the typed error code. For example: "some_example".
Message is the human-readable description of the error.
	Message string `json:"message"`

A CallOption is an optional argument to an API call. It should be treated as an opaque value by users of Google APIs. A CallOption is something that configures an API call in a way that is not specific to that API; for instance, controlling the quota user for an API call is common across many APIs, and is thus a CallOption.
